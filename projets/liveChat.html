Internet Relay Chat : Client & Serveur

Implémentation d’un système de chat en temps réel inspiré d’IRC : un serveur Node.js/ExpressJS + Socket.IO, et un client ReactJS communiquant via WebSocket. Les salons, messages et utilisateurs sont conservés de manière persistante.

Architecture & Composants

  • Serveur : Node.js + ExpressJS expose une API REST et gère les WebSocket via socket.io.
  • Client : ReactJS (hooks/Context) pour l’interface : salons, messages, commandes.
  • Protocol : implémentation simplifiée d’IRC (NICK, JOIN, PRIVMSG, etc.).

Persistance

Les salons et messages sont stockés de façon durable — à l’aide de la méthode choisie (fichiers, base de données, etc.) — pour recharger l’état en cas de redémarrage. Aucune authentification n’est requise, mais peut être ajoutée en bonus.

Commandes disponibles

  • /nick <pseudo> : définit ton surnom.
  • /list [filtre] : liste les salons (optionnellement filtrés).
  • /create <salon> : crée un nouveau canal.
  • /delete <salon> : supprime un canal existant.
  • /join <salon> : rejoint un canal.
  • /quit <salon> : quitte un canal.
  • /users : liste les utilisateurs du canal courant.
  • /msg <pseudo> <message> : message privé à un utilisateur.
  • <message> : envoie un message à tout le canal.

Technologies & Outils

  • Node.js (v18+), ExpressJS, Socket.IO
  • ReactJS (v18), WebSocket natif ou client Socket.IO
  • ESLint, Prettier pour la qualité de code
  • Tests unitaires avec Jest (côté serveur)
